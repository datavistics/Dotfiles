# Dotfiles — Agent Context

This repo manages Derek's shell, editor, and terminal config across machines.
Use this file to understand how the repo works before making changes.

---

## How Symlinks Work

Dotfiles are managed with the [`dotfiles`](https://github.com/jbernard/dotfiles) pip tool.
It symlinks every file in the repo into `~/` with a `.` prefix automatically:

```
Dotfiles/zshrc   →  ~/.zshrc
Dotfiles/vimrc   →  ~/.vimrc
Dotfiles/p10k.zsh → ~/.p10k.zsh
...
```

The config lives in [`dotfilesrc`](dotfilesrc). Files that should **not** be
symlinked (docs, scripts, IDE config) are listed in its `ignore` list.

**To sync all symlinks:**
```bash
dotfiles --sync
```

**To add a new dotfile:**
```bash
cp ~/.newrc ~/Dotfiles/newrc
dotfiles --sync
```
Do not manually `ln -s` — let the tool manage it.

## Fresh Machine Setup

```bash
git clone git@github.com:datavistics/Dotfiles.git ~/Dotfiles
cd ~/Dotfiles && ./install.sh
```

[`install.sh`](install.sh) does in order:
1. `pip install dotfiles` — the symlink manager
2. Installs oh-my-zsh (preserves existing `.zshrc`)
3. Clones `zsh-autosuggestions` into `~/.oh-my-zsh/custom/plugins/`
4. Clones `powerlevel10k` into `~/.oh-my-zsh/custom/themes/`
5. Downloads `vim-plug` to `~/.vim/autoload/plug.vim`
6. Runs `dotfiles --sync --force` to create all symlinks

After install:
- `exec zsh` to reload the shell
- `vim +PlugInstall +qall` to install vim plugins
- `p10k configure` if the prompt needs reconfiguring

## Machine-Specific Config

`~/.zshrc.local` is sourced at the end of `.zshrc` and is **not tracked in git**.
Put anything machine-specific there: conda init blocks, custom PATH entries,
API keys, work aliases, tool-specific setup added by installers.

Common entries:
```zsh
# conda
__conda_setup=...  # generated by `conda init zsh`

# Homebrew (Apple Silicon)
export PATH="/opt/homebrew/bin:$PATH"

# Work tools
export PATH="$HOME/.cargo/bin:$PATH"
export SOPS_AGE_KEY_FILE="$HOME/.config/sops/age/keys.txt"
```

`install.sh` will warn if `~/.zshrc.local` is missing.

## File Map

| File | Symlinked to | Purpose |
|------|-------------|---------|
| `zshrc` | `~/.zshrc` | Main zsh config: oh-my-zsh, vi-mode, fzf, key bindings |

| `p10k.zsh` | `~/.p10k.zsh` | Powerlevel10k prompt config |
| `ideavimrc` | `~/.ideavimrc` | IdeaVim for JetBrains: leader map, EasyMotion, plugins |
| `bashrc` | `~/.bashrc` | Bash fallback config |
| `bash_aliases` | `~/.bash_aliases` | Bash aliases (sourced by bashrc) |
| `bash_options` | `~/.bash_options` | Bash options (sourced by bashrc) |

- `CHEATSHEET.md` — keybinding reference
- `README.md` — repo overview

## Key Design Decisions

**vi-mode everywhere.** `jk` exits insert mode in zsh, vim, and IdeaVim.
Leader key is `Space` in both vim and IdeaVim.

**fzf is central.** Wired into zsh (`^R` history, `^T` file paste, `Alt+C` cd),
into vim (`:Files`, `:Rg`, `:Buffers`), and installed via oh-my-zsh plugin.

**Tmux prefix is `Ctrl-a`**, not the default `Ctrl-b`. Splits use `|` and `-`.
Pane navigation is `prefix + hjkl`. Copy mode uses vi bindings with `pbcopy`.

**No machine-specific code in tracked files.** Everything that varies per
machine belongs in `~/.zshrc.local`. The `dotfiles` tool only tracks the
shared config.

## Adding / Changing Things

**To edit a config** — edit the file in `~/Dotfiles/`, changes reflect
immediately since everything is symlinked.

**To add a new dotfile:**
```bash
cp ~/.<newfile> ~/Dotfiles/<newfile>
dotfiles --sync
git -C ~/Dotfiles add <newfile> && git -C ~/Dotfiles commit -m "Add <newfile>"
git -C ~/Dotfiles push
```

**To add a new zsh plugin** (community plugin from oh-my-zsh):
1. Add the name to `plugins=(...)` in `zshrc`
2. If it's a third-party plugin, add its clone to `install.sh`

**To update p10k prompt config:**
```bash
p10k configure        # runs wizard, writes ~/.p10k.zsh → ~/Dotfiles/p10k.zsh
git -C ~/Dotfiles add p10k.zsh && git -C ~/Dotfiles commit -m "Update p10k config"
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| Shell errors on startup | Check `~/.zshrc.local` for broken paths; make all `source` calls conditional with `[ -f ... ] &&` |
| `p10k` command not found | oh-my-zsh or the theme didn't load; run `./install.sh` |
| Vim plugins missing | Run `:PlugInstall` inside vim |
| Dotfile not symlinked | Run `dotfiles --list` to see status, then `dotfiles --sync` |
| New file showing up in `dotfiles --list` as unwanted | Add it to the `ignore` list in `dotfilesrc` |
